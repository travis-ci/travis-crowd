    Future Plans

    Here are some of the things we plan to do in the near future. Of course there's way more in the pipeline. If you don't see your personal pet feature on this list then don't hesitate to ask!

    • Pre-Tested Pull-Requests

      That shiny green "Merge pull request" button on Github? It's too tempting to click it right? But you don't know if it passes. So you'll have to go back and revert if it breaks. We're going to fix this and pre-test pull requests from Github so you can merge code that passes.

    • Moar languages!

      Travis CI has started out with just Ruby. Today we already support other languages like Node.js, PHP, Erlang and Closure. We want to expand this and provide full multi-version support for Python, Scala, Java and potentially more languages.

    • Build artifacts

      The only information you currently can get back from your tests is the log and the exit code. We want to open this up and allow to upload various types of build artifacts. So you could take screenshots from your app. Or you'd analyse the coverage or code complexity and upload results as files for later use. Or whatever else comes to mind.

    • Private build support

      This is probably the single most often asked for feature on conferences, Twitter, IRC, ... everywhere. People do want to use Travis CI for their private projects. So we are going to make this possible.

    That's right. This stuff is going to rock. We can't wait for these features either :)

    FAQ

    Who are you, actually?

    We've just recently founded a company Travis CI GmbH in Berlin, Germany. So we are able to receive payments, send invoices and pay people in a legal manner.

    This company is currently owned by Josh Kalderimis and Sven Fuchs and will be used to push Travis CI further.

    What are you going to do with all the money?

    We are going to continue work on Travis CI in order to make it even better. Some of the features that we want to work on next are listed above.

    Why do you need to accept donations for this?

    We have been working on Travis CI off our personal money for the last half a year or so and now we have reached a limit with that.

    We are going to implement a paid service for private builds, too, but that will probably take a while. In the meanwtime we want to pay people for improving various parts of the system, too.

    Can't you just sell private build support and use that money for your plans?

    We certainly could. But this would then take much longer as we would need to pick up external work in order to pay our own bills. There also are other features (like pre-tested Github pull requests and the like) that would need to wait even longer. And we believe the community should get them as soon as possible.

    So, put simple, by accepting donations and sponsorships we allow the community to speed up further improvements to Travis CI.
