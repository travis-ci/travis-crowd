    Future Plans

    Here are some of the things we plan to do in the near future. Of course there's way more in the pipeline. If you don't see your personal pet feature on this list then don't hesitate to ask!

    • Pre-Tested Pull-Requests

      That shiny green "Merge pull request" button on Github? It's too tempting to click it right? But you don't know if it passes. So you'll have to go back and revert if it breaks. We're going to fix this and pre-test pull requests from Github so you can merge code that passes.

    • Moar languages!

      Travis CI has started out with just Ruby. Today we already support other languages like Node.js, PHP, Erlang and Closure. We want to expand this and provide full multi-version support for Python, Scala, Java and potentially more languages.

    • Build artifacts

      The only information you currently can get back from your tests is the log and the exit code. We want to open this up and allow to upload various types of build artifacts. So you could take screenshots from your app. Or you'd analyse the coverage or code complexity and upload results as files for later use. Or whatever else comes to mind.

    • Private build support

      This is probably the single most often asked for feature on conferences, Twitter, IRC, ... everywhere. People do want to use Travis CI for their private projects. So we are going to make this possible.

    That's right, this stuff is going to rock. We can't wait for these features either :)
